# 双进程架构说明

## 为什么这样设计？

解决 picamera2 与 mediapipe 的 Python 版本冲突：
- **picamera2**：依赖系统 Python 3.13 + 系统库
- **mediapipe**：需要 Python 3.11（3.13 没有兼容 wheel）

## 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                     系统 Python 3.13                        │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  camera_server.py (摄像头服务器)                      │  │
│  │  - picamera2 取帧                                    │  │
│  │  - 通过 localhost:65432 发送帧                       │  │
│  └──────────────────────┬───────────────────────────────┘  │
│                         │ TCP Socket                        │
└─────────────────────────┼───────────────────────────────────┘
                          │
┌─────────────────────────┼───────────────────────────────────┐
│                  Python 3.11 venv                          │
│  ┌──────────────────────▼───────────────────────────────┐  │
│  │  inference_client.py (推理客户端)                    │  │
│  │  - 接收帧                                            │  │
│  │  - mediapipe 人脸检测                                │  │
│  │  - OpenCV 显示 UI                                    │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 文件说明

| 文件 | 说明 | Python |
|------|------|--------|
| `main.py` | 主启动器，管理两个子进程 | 系统 3.13 |
| `camera_server.py` | 摄像头服务器，负责 picamera2 | 系统 3.13 |
| `inference_client.py` | 推理客户端，负责 mediapipe | 3.11 venv |
| `setup_venv.py` | 设置 Python 3.11 venv | 系统 3.13 |
| `requirements-camera.txt` | 摄像头端依赖 | - |
| `requirements-inference.txt` | 推理端依赖 | - |
| `start.sh` | 一键启动脚本 | - |

## 快速开始

### 1. 设置环境

```bash
python3 setup_venv.py
```

### 2. 启动

```bash
# 方式一：一键启动
./start.sh

# 方式二：手动启动
/usr/bin/python3 main.py
```

### 3. 分别测试（调试用）

终端 1 (摄像头服务器)：
```bash
/usr/bin/python3 camera_server.py
```

终端 2 (推理客户端)：
```bash
venv_311/bin/python inference_client.py
```

## 通信协议

使用简单的 TCP Socket + pickle 序列化：

```
[4字节长度][pickle序列化的numpy数组]
```

## 优点

✅ **兼容性最好** - 两边都用最适合自己的 Python 版本  
✅ **排错简单** - 两个进程独立，可以分别测试  
✅ **稳定可靠** - 升级一边不影响另一边  
✅ **可扩展** - 后续可以替换算法模块，不动相机模块  

## 故障排查

### 端口被占用

如果 `localhost:65432` 被占用，可以修改两个脚本中的 `PORT` 常量。

### venv 创建失败

确保已安装 Python 3.11：
```bash
# 使用 pyenv (推荐)
curl https://pyenv.run | bash
# 重新加载终端后
pyenv install 3.11.9
pyenv global 3.11.9
```
