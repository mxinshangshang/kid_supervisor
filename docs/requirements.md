# 需求规格说明

## 1. 项目目标

系统目标是为儿童书桌学习场景提供一个本地化、非侵入式、可长期运行的监督方案。

核心目标：

1. 检测孩子是否仍在学习位
2. 检测持续性不良姿态
3. 在正面机位下检测过近距离
4. 记录学习时长并提醒休息
5. 在树莓派 5 上长期稳定运行

## 3. 详细需求

### 3.1 存在检测

要求：

1. 连续 `presence_enter_frames` 帧检测到人后，确认进入学习状态
2. 连续 `presence_exit_frames` 帧未检测到人后，确认离开学习状态
3. 无人时暂停当前学习会话

默认值：

1. 进入：2 帧
2. 离开：5 帧

### 3.2 姿态监督

要求：

1. 使用滑动窗口平均评分
2. 姿态问题需持续超过阈值后触发提醒
3. 同类提醒应受冷却时间限制
4. 机位不同，规则应不同

正面机位重点：

1. 低头
2. 歪头
3. 前倾
4. 双肩不平

侧面机位重点：

1. 头前伸
2. 趴桌/贴桌
3. 前倾

### 3.3 距离监督

要求：

1. 仅在正面机位下启用距离提醒
2. 基于头部尺度进行轻量距离估算
3. 使用 EMA 平滑
4. 低置信度读数不应直接触发提醒
5. 边缘区域应降权处理

### 3.4 学习时长提醒

要求：

1. 连续学习超过阈值后提醒休息
2. 进入休息状态后，休息结束给出恢复提醒
3. 会话结束时保存统计数据

### 3.5 自动恢复

要求：

1. 任一子进程异常退出后自动重启
2. 使用退避策略避免频繁重启
3. 子进程稳定运行一段时间后重启计数自动清零

### 3.6 持久化

要求：

1. 至少保存学习会话摘要
2. 数据写入 SQLite
3. 不引入复杂服务依赖

## 5. 约束条件

1. 目标硬件为树莓派 5
2. 单摄像头输入
3. 不依赖云端服务
4. 优先轻量、稳定、可维护方案

---

## 6. 验收重点

验收时重点关注：

1. 正面机位下距离提醒是否基本可信
2. 侧面机位下前倾和趴桌提醒是否明显更稳定
3. 存在检测是否不易抖动
4. 无头模式日志是否持续输出
5. 子进程异常退出后是否能恢复
6. 学习会话是否能写入 SQLite

---

## 7. 当前不纳入范围

当前阶段不纳入：

1. 多摄像头融合
2. Web 管理后台
3. 云端识别
4. 重型模型替换
5. 完整语音交互

原因：这些内容不符合当前版本“树莓派 5 单机稳定监督”的主要目标。
