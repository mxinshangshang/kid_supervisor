# 需求规格说明

## 1. 项目背景与目标

### 1.1 项目背景

随着儿童使用电子设备学习的时间增加，不良的学习姿势和过近的观看距离成为影响儿童视力和脊柱健康的重要因素。本项目旨在通过机器视觉技术，提供一个非侵入式的学习监督方案。

### 1.2 项目目标

- 实时检测儿童学习时的姿势和距离
- 对不良姿势和过近距离给出及时提醒
- 记录学习时长，提醒合理休息
- 本地处理，保护儿童隐私

## 3. 功能详细说明

### 3.1 人脸检测与存在感知

**功能描述**：检测摄像头视野内是否有人脸，判断是否有人在学习。

**详细需求**：
- 支持 640x480 分辨率输入
- 检测频率：最高 20 FPS
- 防抖机制：检测到人脸 2 帧后确认存在，消失 3 帧后确认离开
- 支持无人状态自动暂停计时

### 3.2 学习时长统计

**功能描述**：记录连续学习的时间长度。

**详细需求**：
- 自动开始/暂停计时（基于人脸存在检测）
- 单次学习最大时长：45 分钟（可配置）
- 休息时长：10 分钟（可配置）
- 记录单次学习中的不良姿势和过近距离次数

### 3.3 姿态检测与分析

**功能描述**：检测人体姿态，判断是否存在不良姿势。

**详细需求**：
- 检测的不良姿势包括：
  - 低头/驼背
  - 肩膀不平
  - 身体前倾/趴着
- 不良姿势持续超过 5 秒触发提醒（可配置）
- 提醒冷却时间：30 秒（可配置）

### 3.4 距离估算

**功能描述**：估算人脸与摄像头的距离。

**详细需求**：
- 基于人脸大小的相似三角形原理估算
- 过近距离阈值：35cm（可配置）
- 过近状态持续超过 3 秒触发提醒（可配置）

### 3.5 智能提醒

**功能描述**：对检测到的不良状态给出提醒。

**详细需求**：
- 实时在预览画面显示提醒信息
- 提醒类型包括：
  - 姿态不良（POSTURE_BAD）
  - 距离过近（TOO_CLOSE）
  - 需要休息（BREAK_NEEDED）
  - 休息结束（BREAK_OVER）

## 4. 约束条件

### 4.1 硬件约束

- 主控制器：树莓派 5
- 内存：>= 4GB（推荐 8GB）
- 摄像头：Camera Module 3 兼容摄像头

### 4.2 软件约束

- 操作系统：Raspberry Pi OS (Debian)
- 系统 Python：3.13（用于 picamera2）
- 推理 Python：3.11（用于 mediapipe）

### 4.3 性能约束

- 总体帧率：>= 10 FPS
- 单次推理延迟：< 100ms
- 内存占用：< 500MB

## 5. 验收标准

### 5.1 功能验收

- [ ] 人脸检测准确率 > 90%
- [ ] 姿态检测能识别预设的不良姿势
- [ ] 距离估算误差 < 20%
- [ ] 提醒机制按预期工作

### 5.2 性能验收

- [ ] 系统稳定运行 > 4 小时无崩溃
- [ ] 实时预览帧率 >= 10 FPS
- [ ] 内存占用保持在合理范围
