## 2. 关键技术决策记录

### 2.1 检测引擎选择

**问题**：选择什么样的检测方案？

**选项对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| Haar Cascade | 轻量，系统自带 | 精度低，特别是对姿态 |
| MediaPipe Face Mesh | 精度高，人脸关键点全 | 无法分析全身姿态 |
| MediaPipe Pose | 全身 33 个关键点，可分析姿态 | 计算量稍大 |
| MoveNet + TFLite | 轻量，针对移动设备优化 | 需更多适配工作 |

**决策**：选择 **MediaPipe Pose**

- 树莓派 5 8GB 性能足够
- 姿态分析是核心需求
- MediaPipe 有预编译包（Python 3.11）

### 2.2 Python 版本冲突解决方案

**问题**：
- picamera2 仅支持系统 Python 3.13
- mediapipe 没有 Python 3.13 的预编译 wheel

**尝试过的方案**：

1. **源码编译 MediaPipe for 3.13**
   - 结果：依赖复杂，编译失败，放弃

2. **换用其他检测库**
   - 结果：要么精度不够，要么同样有版本问题

3. **双进程架构**
   - 结果：验证可行，最终采用

**最终方案**：双进程 + Socket 通信

## 3. 调试问题与解决方案

### 3.1 颜色空间问题

**问题现象**：
- 画面颜色异常，肤色偏紫/偏红

**原因分析**：
- Picamera2 输出 RGB888
- OpenCV 期望 BGR
- 直接传递导致颜色通道错位

**解决方案**：
```python
# 接收 RGB 帧后，渲染时转换为 BGR
frame_bgr = cv2.cvtColor(frame_rgb, cv2.COLOR_RGB2BGR)
```

### 3.2 GLib-GObject 警告

**问题现象**：
```
GLib-GObject-CRITICAL **: ...: assertion 'G_IS_OBJECT (object)' failed
```

**原因分析**：
- OpenCV 窗口销毁时的清理顺序问题
- 不影响核心功能

**缓解方案**：
```python
try:
    cv2.destroyWindow(window_name)
    cv2.destroyAllWindows()
except Exception:
    pass  # 忽略清理错误
```

### 3.3 人脸检测抖动

**问题现象**：
- 存在检测频繁跳变，导致计时频繁启停

**原因分析**：
- 检测不是 100% 可靠，偶尔漏检
- 无防抖机制

**解决方案**：
```python
# 检测到人：2 帧确认
# 人离开：3 帧确认
person_counter = min(person_counter + 1, 3)
if person_counter >= 2 and not person_detected:
    # 确认存在
person_gone_counter = min(person_gone_counter + 1, 5)
if person_gone_counter >= 3 and person_detected:
    # 确认离开
```

### 3.4 提醒过于频繁

**问题现象**：
- 同一问题反复提醒，打扰用户

**解决方案**：
- 增加提醒冷却机制（30 秒）
- 需要不良状态持续一段时间才提醒（姿态 5 秒，距离 3 秒）

### 3.5 OpenCV 中文显示问号

**问题现象**：
- 预览界面上的中文提示显示为问号（?）

**原因分析**：
- OpenCV 的 `cv2.putText()` 函数默认只支持 ASCII 字符集
- 不支持 UTF-8 中文字符渲染

**解决方案**：
- 将所有界面上的提示文本从中文改为英文
- "肩膀不平" → "Uneven Shoulders"
- "低头/驼背" → "Head Down/Slouching"
- "身体前倾/趴着" → "Leaning Forward"

### 3.6 姿态检测过于敏感，误报多

**问题现象**：
- 用户已经坐得比较正，但仍频繁提示姿势不对
- 侧身机位时检测效果特别差

**原因分析**：
- 判断阈值设置过于严格
- 没有考虑关键点可见性问题
- 没有区分正面朝向和侧身朝向

**解决方案**：
1. **增加可见性检查**：只对 visibility > 0.5 的关键点进行检测
2. **正面/侧身检测**：根据耳朵可见性判断朝向，采用不同策略
3. **放宽阈值**：
   - 肩膀不平：0.05h → 0.08h
   - 低头：0.03h → 0.07h
   - 身体前倾：0.15h → 0.25h
4. **延长判定时间**：
   - 姿态不良：5秒 → 8秒
   - 距离过近：3秒 → 5秒
5. **增加冷却时间**：30秒 → 45秒
6. **降低距离阈值**：35cm → 30cm

### 3.7 距离读数固定为 75cm，波动大

**问题现象**：
- 距离读数一直是 75cm 不变
- 或者读数波动剧烈，不稳定

**原因分析**：
- 焦距参数 `camera_focal_length` 不适合当前摄像头
- 没有平滑滤波，原始检测抖动大

**解决方案**：
1. **调整默认焦距**：600 → 800
2. **增加平滑滤波**：移动平均滤波，alpha=0.3
3. **限制合理范围**：30cm-150cm 之间
4. **新增校准工具**：`calibrate_distance.py` 支持用户自定义校准

## 5. 已知限制

1. **光线依赖**：MediaPipe 在光线不足时检测率下降
2. **角度支持改进**：v3.1 版本已支持正面和侧身机位，但极端角度仍可能有问题
3. **距离估算精度**：基于人脸大小的估算有一定误差，使用前请用校准工具校准
4. **无自动重启**：推理进程意外退出后需要手动重启
5. **无数据持久化**：当前版本不保存历史记录
