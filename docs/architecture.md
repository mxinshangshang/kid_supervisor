# 架构设计文档

## 1. 架构目标

系统面向树莓派 5 单摄像头儿童书桌监督场景，架构目标为：

1. 解决 `picamera2` 与 `mediapipe` 的 Python 版本冲突
2. 支持长期稳定运行
3. 以低误报、可解释为优先，而不是追求复杂模型堆叠
4. 支持正面和侧面两种常见安装方式

---

## 2. 总体架构

```text
config.yaml
   |
   +-- main.py
   |     +-- camera_server.py      (system python / picamera2)
   |     +-- inference_client.py   (python 3.11 / mediapipe)
   |
   +-- calibrate_distance.py
```

### 2.1 主启动器 `main.py`

职责：

1. 加载统一配置
2. 启动摄像头子进程与推理子进程
3. 监控子进程存活状态
4. 处理优雅退出

### 2.2 摄像头进程 `camera_server.py`

职责：

1. 初始化 `Picamera2`
2. 按配置采集 RGB888 帧
3. 转 BGR 后做 JPEG 编码
4. 通过 TCP Socket 发送帧数据
5. 输出真实发送统计

协议：

```text
[4B frame_id][8B timestamp][4B jpeg_len][jpeg_data]
```

### 2.3 推理进程 `inference_client.py`

职责：

1. 独立接收线程持续收帧
2. 主循环按节拍进行推理、渲染、温控和日志输出
3. 调用姿态/距离检测器
4. 调用监督状态机生成告警
5. 调用通知器输出告警
6. 在会话结束时保存 SQLite 记录

---

## 3. 模块设计

### 3.1 配置模块 `src/config.py`

职责：

1. 提供默认配置
2. 读取 `config.yaml`
3. 深度合并用户配置与默认值
4. 校验关键配置范围

### 3.2 检测模块 `src/vision/pose_detector.py`

职责：

1. 运行 MediaPipe Pose
2. 生成 `DetectionResult`
3. 估算头部框范围
4. 正面模式下进行距离估算
5. 根据机位模式输出不同姿态问题

正面模式重点：

1. 距离
2. 低头
3. 歪头
4. 前倾
5. 双肩不平

侧面模式重点：

1. 头前伸
2. 趴桌/贴桌趋势
3. 前倾

### 3.3 监督模块 `src/supervision.py`

职责：

1. 学习状态管理
2. 滑动窗口姿态评分
3. 距离持续时间判断
4. 休息提醒与恢复提醒
5. 会话历史缓存

### 3.4 通知模块 `src/notifier.py`

职责：

1. 接收统一 `Alert` 对象
2. 输出控制台提醒
3. 预留音频提醒注入接口

### 3.5 预览模块 `src/preview_renderer.py`

职责：

1. 渲染 RGB 预览帧
2. 绘制骨架、人脸框、姿态问题和告警横幅
3. 仅呈现核心监督信息

### 3.6 持久化模块 `src/storage.py`

职责：

1. 使用 SQLite 保存学习会话
2. 为后续统计和回看提供基础数据

## 4. 关键设计决策

### 4.1 接收线程与主循环解耦

原因：

1. 避免阻塞接收导致预览卡顿
2. 保证温控检查和退出响应不被网络节拍绑死

### 4.2 机位模式显式配置

原因：

1. 正面与侧面场景的监督重点不同
2. 一套规则强适配所有机位容易产生误报

### 4.3 轻量距离估算而非重型人脸模型

原因：

1. 贴合树莓派 5 性能边界
2. 先保证稳定和解释性
3. 后续若需要更高精度，可替换为轻量真实人脸框检测

### 4.4 最小持久化优先

原因：

1. 先形成业务闭环
2. 不在当前阶段引入 Web 后台等非核心复杂度

---

## 5. 运行流程

### 5.1 摄像头链路

1. `camera_server.py` 采集 RGB 帧
2. 转 BGR 做 JPEG 编码
3. 发送 header + jpeg 数据

### 5.2 推理链路

1. 接收线程持续拉取最新帧
2. 主循环按 `inference_fps` 推理
3. 生成 `DetectionResult`
4. `Supervisor` 更新状态并产出 `Alert`
5. `Notifier` 输出告警
6. `PreviewRenderer` 按 `display_fps` 更新预览

### 5.3 会话链路

1. 连续检测到人进入学习状态
2. 人离开时结束当前会话
3. 休息提醒触发时也会截断当前会话
4. 会话写入 SQLite

---

## 6. 当前架构边界

当前版本不覆盖：

1. 多摄像头融合
2. 云端推理
3. Web 管理端
4. 重型检测模型替换全链路

这些能力都可能引入超出当前硬件条件的复杂度，不属于本轮架构目标。
